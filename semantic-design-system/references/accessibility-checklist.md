# Accessibility Checklist for Design Tokens

Validate at minimum:

- Text foreground/background contrast.
- Focus-visible ring contrast and visibility.
- Disabled state distinguishability without relying on opacity alone.
- Minimum touch target size, usually represented as `semantic.size.touch-target.min`.
- Reduced-motion behavior for transitions and animations.
- High-contrast mode readiness.
- Error, warning, success, and info states not relying solely on color.
- Form invalid/focus/disabled/read-only state coverage.
- Token metadata records intended foreground/background pairings where contrast matters.
